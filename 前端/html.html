<script>最好放底部，<link>最好放头部，同时有<script>与<link>的情况下，最好将<script>放在<link>上面

  • CSS 不会阻塞 DOM 的解析，但会阻塞 DOM 渲染。

  • JS 阻塞 DOM 解析，但浏览器会"偷看"DOM，预先下载相关资源。

  • 浏览器遇到 <script>且没有defer或async属性的 标签时，会触发页面渲染，因而如果前面CSS资源尚未加载完毕时，浏览器会等待它加载完毕在执行脚本。

_ blank隐藏漏洞

window.parent对象从iframe的页面中访问父页面的window对象。在<iframe>中，提供了一个 sandbox 属性来控制 iframe 中页面的权限，因此即使在同一个域中，也可以控制<iframe>的安全性。

使用window.opener从通过<a target="_blank">打开的页面中访问源页面的window对象。

html
if(window.opener){
	//替换连接钓鱼
    window.opener.location = "链接的网页地址";
}

<a href="https://an.evil.site" target="_blank" rel="noopener noreferrer nofollow">Enter an "evil" website</a>
rel 属性用于指定当前文档与被链接文档的关系。
noopener noreferrer 在打开的新选项卡中，opener 对象将不再可用，因为它已被设置为 null。(一些老旧浏览器不支持noopener)
nofollow 不想将权重传递给第三方链接或不重要或不希望谷歌索引的页面

    • column-reverse

  • justify-content设置主轴方向子元素排列方式

    • flex-start
    • flex-end
    • center
    • space-around
    • space-between先左右贴边，再分配剩余空间

  • flex-wrap设置子元素是否换行

  • align-content设置侧轴方向子元素排列方式（多行）

    • flex-start

htmx ​

Htmx （X应该是 extension，是“扩展”的意义。）提供了一种通过 HTML（而不是 JavaScript）访问各种浏览器特性的方式。

HATEOAS

Hypermedia as the Engine of Application State的缩写，即超媒体驱动应用状态。每次应用程序状态的变化，不是靠数据驱动，而是页面结构和元素（超媒体）直接驱动。即让 server 返回下一个状态的 HTML 结构，响应结果可以直接嵌入当前 HTML 页面的某个位置，让应用程序进入下一个状态，即直接通过超媒体驱动，省去了大量 JavaScript 的操作。

html
<button hx-post="/clicked"
    hx-trigger="click"
    hx-target="#parent-div"
    hx-swap="outerHTML"
>Click Me!</button>
<!-- 当用户单击按钮，向/click的路由发送POST请求，并把返回的内容设置到#parent-div表示的容器中。 -->
