Python ​

Python的语言实现：

**1.CPython：**这是Python的官方版本，使用C语言实现，使用最为广泛，新的语言特性一般也最先出现在这里。CPython实现会将源文件（py文件）转换成字节码文件（pyc文件），然后运行在Python虚拟机上。

**2.Jython：**这是Python的Java实现，相比于CPython，它与Java语言之间的互操作性要远远高于CPython和C语言之间的互操作性。

**3.Python for .NET：**它实质上是CPython实现的.NET托管版本，它与.NET库和程序代码有很好的互操作性。

4.**PyPy：**Python的Python实现版本，原理是这样的，PyPy运行在CPython（或者其它实现）之上，用户程序运行在PyPy之上。它的一个目标是成为Python语言自身的试验场，因为可以很容易地修改PyPy解释器的实现（因为它是使用Python写的）。

gini系数 ​

洛伦兹曲线

洛伦兹曲线研究的国民收入在国民之间的分配问题。它是美国统计学家洛伦兹提出的。将人口累计百分比和收入累计百分比的对应关系描绘在图形上，即得到洛伦兹曲线，ODL为该图的洛伦兹曲线。收入分配完全平等的洛伦兹曲线成为通过原点的45°线OL。收入分配达到完全不平等，洛伦兹曲线成为折线OHL。

G为基尼系数，则：G=A/(A+B)(0≤G≤1)。基尼系数被西方经济学家普遍公认为一种反映收入分配平等程度的方法。基尼系数最大为“1”，最小等于“0”。基尼系数越接近0表明收入分配越是趋向平等。

gini系数用来评估一个集合的纯度：基尼系数实际上就是计算从集合中随意抽2个样本（有放回的），这2个样本不是同一类的概率。基尼系数越小，集合纯度越高。如果集合中的样本只有一个类别，那么基尼系数就是0，集合的纯度最高。Gini指数是衡量数据的不均衡程度的一种指标，可以用来评估分类模型的效果。

python
import numpy as np


def gini(data_list):
    """获得列表中的gini系数"""
    data_length = len(data_list)
    total_sum = np.sum(data_list)
    total_gini = 0
    for i in range(data_length):
        temp_denominator = data_list[i][0] + data_list[i][1]
        temp_son = np.sum(data_list[i])
        temp_gini = 1 - np.power(data_list[i][0] / temp_denominator, 2) - np.power(data_list[i][1] / temp_denominator,
                                                                                   2)
        total_gini = temp_gini * temp_son / total_sum + total_gini
    return total_gini


if __name__ == '__main__':
    data = [[2, 3], [2, 7]]
    print(gini(data))

代价函数

数据集(训练集)中的每个样本都有相应的“正确答案”。再根据这些样本作出预测。

模型所预测的值与训练集中实际值之间的差距就是建模误差（modeling error）。可以使得建模误差的平方和能够最小的模型参数。 即使得代价函数最小，也被称作平方误差函数，有时也被称为平方误差代价函数。

$$ J(\theta_0,\theta_1) = \frac{1}{2m}\sum^m_{i=1}(h_\theta(x^i)-y^i)^2 $$

梯度下降

梯度下降是一个用来求函数最小值的算法。开始时我们随机选择一个参数的组合，计算代价函数，然后我们寻找下一个能让代价函数值下降最多的参数组合。我们持续这么做直到到到一个局部最小值。当我们接近局部最低点时，梯度下降法会自动采取更小的幅度。

批量梯度下降（batch gradient descent）算法的公式为：

$$ \theta_j:=\theta_j-\alpha\frac{∂}{∂\theta_j}J(\theta_0,\theta_1) $$

α是学习率（learning rate），它决定了我们沿着能让代价函数下降程度最大的方向向下迈出的步子有多大。在批量梯度下降中，我们每一次都同时让所有的参数减去学习速率乘以代价函数的导数。

梯度下降算法的每次迭代受到学习率的影响，如果学习率过小，则达到收敛所需的迭代次数会非常高；如果学习率过大，每次迭代可能不会减小代价函数，可能会越过局部最小值导致无法收敛。通常可以考虑尝试些学习率：$\alpha = 0.01,0.03,0.1,0.3,1,3,10$

除了梯度下降算法以外，还有一些常被用来令代价函数最小的算法，这些算法更加复杂和优越，而且通常不需要人工选择学习率，有一个智能的内部循环，称为线性搜索(line search)算法，它可以自动尝试不同的学习速率 ，并自动选择一个好的学习速率，通常比梯度下降算法要更加快速。这些算法有：共轭梯度（Conjugate Gradient），局部优化法(Broyden fletcher goldfarb shann,BFGS)和有限内存局部优化法(LBFGS)

特征缩放

面对多维特征问题的时候，我们要保证这些特征都具有相近的尺度，这将帮助梯度下降算法更快地收敛。解决的方法是尝试将所有特征的尺度都尽量缩放到-1到1之间。最简单的方法是令：$x_n = \frac{x_n - \mu _n}{S_n}$，其中n是平均值，sn是标准差。

正规方程

正规方程是通过求解下面的方程来找出使得代价函数最小的参数的：

$$ \frac{∂}{∂\theta_j}J(\theta_j) = 0 \ \theta = (X^TX)^{-1}X^Ty $$

梯度下降正规方程
需要选择学习率不需要
需要多次迭代一次运算得出
当特征数量大时也能较好适用需要计算 如果特征数量n较大则运算代价大，因为矩阵逆的计算时间复杂度为$O(n^3)$，通常来说当小于10000 时还是可以接受的
适用于各种类型的模型只适用于线性模型，不适合逻辑回归模型等其他模型

单变量线性回归问题

$$ h_\theta(x) = \theta_0 + \theta_1x $$

梯度下降的线性回归：

$$ \theta_0:=\theta0 - \alpha\frac{1}{m}\sum^m_{i=1}(h_\theta(x^i) - y^i) \ \theta_1:=\theta1 - \alpha\frac{1}{m}\sum^m_{i=1}((h_\theta(x^i) - y^i) * x^i) $$

多变量线性回归

$$ h_\theta(x) = \theta^TX = \theta_0x_0 + \theta_1x_1 + \cdots + \theta_nx_n \qquad (x_0 = 1) $$

多项式回归

$$ h_\theta(x) = \theta_0x_0 + \theta_1x_1 + \theta_2x_2^2 + \theta_3x_3^3 \qquad (x_0 = 1) $$

逻辑回归 ​

推测出离散的输出值

  • 线性分类器
  • 非线性分类器

二元分类

该模型的输出变量范围始终在0和1之间。 逻辑回归模型的假设是

$$ h_\theta(x) = g(\theta^TX) \ g(z) = \frac{1}{1+e^{-z}} \ h_\theta(x) = \frac{1}{1+e^{-\theta^TX}} $$

对逻辑回归模型沿用误差的平方和得到的代价函数将是一个非凸函数，这意味着我们的代价函数有许多局部最小值，这将影响梯度下降算法寻找全局最小值。

$$ J(\theta) = \frac{1}{m}\sum^m_{i=1}Cost(h_\theta(x^i)-y^i) \ Cost(h_\theta(x^i)-y^i) = \begin{cases} -log(h_\theta(x)) \ if \ y = 1 \ -log(1 - h_\theta(x)) \ if \ y = 1 \ \end{cases} $$

正则化 ​

欠拟合，不能很好地适应我们的训练集；过拟合，过于强调拟合原始数据，而丢失了算法的本质，可以丢弃一些不能帮助我们正确预测的特征。可以是手工选择保留哪些特征，或者使用一些模型选择的算法来帮忙（例如PCA）。

正则化保留所有的特征，但是减少参数的大小，可以改善或者减少过度拟合问题。每次都在原有算法更新规则的基础上令$\theta$值减少了一个额外的值。

$$ 正则化的线性回归 \qquad \lambda为正则化参数,\theta_0不参与其中的任何一个正则化 \ J(\theta) = \frac{1}{2m}[\sum^m_{i=1}(h_\theta(x^i)-y^i)^2 + \lambda\sum^n_{j=1}\theta_j^2] \正则化的逻辑回归 \ J(\theta) = \frac{1}{m}\sum^m_{i=1}[-y^ilog(h_\theta(x^i)) -(1-y^i)log(1 - h_\theta(x^i))] + \frac{\lambda}{2m}\sum^n_{j=1}\theta_j^2 $$

学习模型保存加载使用

python机器学习模型训练完之后可以保存为pkl文件，也可以保存为pmml文件。给python开发用就保存为pkl文件，给java开发用就保存为pmml文件。

机器学习模型保存为pkl文件有两种方式：可以使用joblib包，也可以使用pickle包。joblib包是由scikit-learn外带的，常用于保存机器学习模型。对于大数据而言，joblib比pickle更加高效。

python
# 使用joblib保存模型为pkl文件
import joblib
# 参数model：要保存的模型（fit训练后的）。
# 参数filename：要保存的模型路径和名称。
joblib.dump(model, filename='./model_joblib.pkl')
model = joblib.load(filename='./model_joblib.pkl')

# 使用pickle保存模型为pkl文件
import pickle
pickle.dump(model, file=open('./model_pickle.pkl', 'wb+'))
model = pickle.load(file=open('./model_pickle.pkl', 'rb'))

计算权重 ​

结合各类方法计算权重的原理大致上可分成4类，分别如下：

第一类为因子分析和主成分法；此类方法利用了数据的信息浓缩原理，利用方差解释率进行权重计算；因子分析法和主成分法的区别在于，因子分析法加带了‘旋转’的功能，而主成分法目的更多是浓缩信息。

第二类为AHP层次法和优序图法；此类方法利用数字的相对大小信息进行权重计算；

第三类为熵值法（熵权法）；此类方法利用数据熵值信息即信息量大小进行权重计算；

第四类为CRITIC、独立性权重和信息量权重；此类方法主要是利用数据的波动性或者数据之间的相关关系情况进行权重计算。

Prophet ​

Prophet 是一个 Python 的微框架，用于金融市场。Prophet 可以让开发人员把精力放在金融策略模型、项目组合管理和分析上。

Prophet模型将时间序列分解为四个主要的成分：趋势、季节性、假日效应和噪声。其中，趋势是时间序列的长期变化趋势，季节性是周期性变化的模式，假日效应是指在特殊日期或时间段内的影响，噪声则是无法被预测的随机变化。Prophet模型既可以使用加法模型，也可以使用乘法模型。

$$ y(t)=g(t)+s(t)+h(t)+e(t) \ y(t)=g(t)*s(t)*h(t)*e(t) $$

其中，g(t)表示时间序列的趋势，s(t)表示时间序列的季节性，(t)表示时间序列的假期效应，(t)表示模型的误差项。在加法模型中，趋势、季节性和假期效应都是原始值，并且都被解释为相对于时间序列的平均水平的偏差。

通常情况下，加法模型适用于时间序列的趋势和季节性与数据规模无关的情况，例如气温和降雨量；而乘法模型适用于时间序列的趋势和季节性与数据规模相关的情况，例如商品销售量和股票价格。

ARIMA ​

ARIMA（差分自回归移动平均模型）根据已有的历史时间数据，预测未来的趋势走向（时间序列建模）。

适用条件

  • 数据序列是平稳的，这意味着均值和方差不应随时间而变化。通过对数变换或差分可以使序列平稳。
  • 输入的数据必须是单变量序列，因为ARIMA利用过去的数值来预测未来的数值
